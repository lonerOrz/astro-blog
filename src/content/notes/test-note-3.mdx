### 🛠️ 核心代码快照

为了让 Astro 的语法高亮插件（如 Shiki）承受真实解析压力，以下嵌入一段高内聚的 TypeScript 泛型推导代码：

```typescript
// 模拟高阶类型推导负载
type DynamicModule<T extends string> = {
  [K in T]: () => Promise<{ default: any }>;
};

export class Orchestrator<T extends string> {
  private registry: Map<string, any> = new Map();

  async bootstrap(modules: DynamicModule<T>): Promise<void> {
    const entries = Object.entries(modules);
    await Promise.all(entries.map(async ([key, load]) => {
      const mod = await load();
      this.registry.set(key, mod.default);
    }));
  }
}
```

> 💡 **设计备忘：** 当 `var(--order)` 的阶梯延迟乘以基数时，第 100 篇笔记的动画触发延迟将达到 `calc(100 * 0.04s)`。在卡片总数极高时，应当引入 **虚拟列表（Virtual List）** 或 **滚动渐显（Intersection Observer）** 来释放渲染压力。
