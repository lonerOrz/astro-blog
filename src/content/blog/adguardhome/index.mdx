## 什么是双 AdGuardHome 分流

双 AdGuardHome（双 ADG）通常用于：

- 国内 DNS 分流
- 国外 DNS 分流
- 广告过滤
- 提升 DNS 解析速度
- 配合 PassWall / OpenClash 使用

常见方案：

- 一个 ADG 负责国内 DNS
- 一个 ADG 负责国外 DNS
- OpenWrt 根据规则转发

<Callout>
  本教程使用 Docker 部署 AdGuardHome。
</Callout>

---

## Docker 部署 AdGuardHome

### 启动容器

```bash
docker run -d --name adg \
  -v /mnt/usb1-1/docker/AGH_Docker/work:/opt/adguardhome/work \
  -v /mnt/usb1-1/docker/AGH_Docker/conf:/opt/adguardhome/conf \
  -p 3001:3000 \
  -p 5336:5336 \
  --restart always \
  adguard/adguardhome:v0.107.46
```

参数说明：

- `3001:3000`
  - Web 管理界面
- `5336:5336`
  - DNS 服务端口
- `work`
  - 工作目录
- `conf`
  - 配置目录

## OpenWrt 软路由配置

### DHCP / DNS 设置

进入：

```txt
网络 → DHCP/DNS → DNS 转发
```

设置：

```txt
127.0.0.1#5334
```

说明：

- `5334`
  - 国内 DNS 端口
- `127.0.0.1`
  - 本机 DNS

---

## PassWall 配置

进入：

```txt
PassWall → DNS → 远程 DNS
```

设置：

```txt
172.17.0.2:5336
```

说明：

- `172.17.0.2`
  - Docker 容器 IP
- `5336`
  - 国外 DNS 端口

如果你的 Docker 网段不同，请修改为实际 IP。

---

## 推荐网络结构

```txt
客户端
   ↓
OpenWrt
   ↓
国内域名 → 国内 ADG
国外域名 → 国外 ADG
```

这样可以：

- 减少 DNS 污染
- 提升解析速度
- 提高广告过滤效率
- 避免国内 DNS 泄露

## 常见问题

### Docker 容器无法访问网络

检查：

```bash
docker ps
docker network ls
```

---

### AdGuardHome 无法启动

检查端口占用：

```bash
sudo lsof -i :5336
```

---

### DNS 不生效

检查：

- OpenWrt DNS 转发
- PassWall DNS 设置
- Docker 网络模式

双 AdGuardHome 分流方案适用于：

- 国内外 DNS 分离

相比单 DNS：
